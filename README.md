## MLOps Pipeline: Titanic Dataset

# Цель проекта


Демонстрация работы полноценного MLOps-пайплайна. Проект включает в себя автоматизацию подготовки данных, обучение модели с логированием экспериментов, версионирование данных и создание хранилища признаков (Feature Store).

## Как запустить

Для воспроизведения проекта выполните следующие команды в терминале:


``` bash

pip install -r requirements.txt
chmod +x setup_dvc.sh && ./setup_dvc.sh
dvc repro
mlflow ui --backend-store-uri sqlite:///mlflow.db

```

## Краткое описание пайплайна

- Проект организован как направленный ациклический граф (DAG) и управляется через dvc.yaml:

- Stage: prepare — Скрипт src/split_data.py разделяет исходный датасет data/titanic.csv на обучающую и тестовую выборки.

- Stage: train — Скрипт src/train.py обучает модель, используя гиперпараметры из params.yaml, и фиксирует результаты в MLflow.

- Data Versioning — Все файлы в папке data/ отслеживаются через DVC и не попадают в Git-репозиторий, обеспечивая чистоту кода.

- Feature Store — В папке my_feature_repo настроен Feast для централизованного управления признаками.

## Где смотреть UI MLflow

После запуска команды mlflow ui интерфейс со всеми графиками, параметрами и метрикой accuracy будет доступен в вашем браузере по адресу: http://127.0.0.1:5000
